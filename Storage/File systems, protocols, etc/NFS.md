Network File System (NFS)

Network File System (NFS) es un protocolo de red que permite compartir archivos y carpetas a través de una red, de forma que otros equipos puedan acceder a ellos como si fueran parte de su propio sistema de archivos.

Una de sus principales ventajas es que no está limitado a Windows. NFS es muy usado en Linux y Unix, pero también puede integrarse con Windows, lo que lo hace ideal para entornos mixtos donde conviven distintos sistemas operativos.

# ¿Cómo funciona NFS?

Un equipo actúa como servidor NFS, donde se almacenan los archivos.

Otros equipos funcionan como clientes, que se conectan al servidor.

Los clientes “montan” las carpetas compartidas y pueden leer, escribir, crear o eliminar archivos, según los permisos asignados.

Todo este proceso se realiza de forma transparente para el usuario, como si los archivos estuvieran en el disco local.

# Acceso multiprotocolo (SMB y NFS)

Es posible configurar un servidor de archivos para ofrecer acceso multiprotocolo, es decir:

SMB (Server Message Block) para clientes Windows.

NFS para clientes Linux/Unix.

Esto permite que los mismos archivos y carpetas puedan ser accedidos desde distintos sistemas operativos sin duplicar datos, facilitando la interoperabilidad en redes empresariales o educativas.

# Ventajas de NFS

Compatibilidad con múltiples sistemas operativos.

Ideal para redes Linux/Unix.

Permite compartir archivos de forma centralizada.

Puede coexistir con SMB en un mismo servidor.

# Uso común de NFS

Servidores Linux y Unix.

Entornos de virtualización.

Laboratorios, universidades y centros de datos.

Redes donde se requiere compatibilidad entre Windows y Linux.
